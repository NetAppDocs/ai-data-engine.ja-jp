= AI Data Engineワークスペースへのユーザーアクセスの割り当て
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ストレージ管理者は、組織の構造とニーズに応じて、データ エンジニア、データ サイエンティスト、その他の役割に基づいてユーザーをワークスペースに割り当てます。ユーザーは資格情報を使用してAI Data Engine（AIDE）コンソールにログインし、割り当てられたワークスペース内のデータ コンテナ リソースにアクセスします。

ONTAP System Managerを使用すると、AI Data Engineワークスペースにアクセスできるユーザーを管理できます。ユーザーを追加または削除して、ワークスペースのデータとアクティビティを表示、変更、操作できるユーザーを制御できます。

.開始する前に
* ワークスペースへのユーザー アクセスを管理するには、_storage administrator_ 権限が必要です。
* ワークスペースが作成され、Workspaces インベントリ内でアクティブになっていることを確認します。
* 関連するすべてのデータ コンテナがワークスペースに追加され、アクセス可能であることを確認します。
* link:../install-setup/configure-oidc.html["クラスタで OIDC が有効化され、設定されていることを確認します"]。IdP から ONTAP ロールへのロール マッピングは、関連する各データ エンジニアおよびデータ サイエンティストの IdP ユーザーまたはグループに対して完了する必要があります。


.手順
* ワークスペースにユーザーを追加します：
+
.. ONTAP System Managerで、*Data Engine > ワークスペース*に移動します。
.. ワークスペース名を選択して詳細ページを開きます。
.. 「ユーザー」タブに移動します。
.. *追加* ボタンを選択して、ユーザー追加ダイアログを開きます。
.. 1 人以上のユーザーの詳細を入力します。OIDC ユーザーのコンマ区切りリストとして詳細を入力します。
.. ユーザーにワークスペースへのアクセスを許可するには、*追加*を選択します。


* ワークスペースからユーザーを削除します：
+
.. ワークスペースの詳細ページの「ユーザー」タブで、削除するユーザーを見つけて選択します。
.. *削除* ボタンを選択します。
.. ダイアログで削除を確認します。
.. システムは直ちにユーザーを削除し、ワークスペースへのアクセスを取り消します。




.結果
ワークスペースの [ユーザー] タブにリストされているユーザーのみが、ワークスペースのデータとアクティビティにアクセスして操作できます。
