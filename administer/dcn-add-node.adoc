= データ計算ノードをAIDE クラスタに追加
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
新しいAI Data Engine（AIDE）クラスタの作成時または既存のクラスタの拡張時に、データコンピュートノード（DCN）を追加できます。ワークフローは、ONTAP System Managerを使用したノードの検出と設定で構成されます。



== ノードを追加する準備

DCNノードを追加する際にはいくつかの考慮事項があります。

.新しいAIDEクラスタを作成する場合
新しいAIDEクラスタには、ちょうど3つのDCNノードが必要です。

.ハードウェアのインストールとアドレス指定
次の前提条件が満たされていることを確認してください：

* 新しいDCNハードウェアがラックに設置され、電源がオンになり、クラスタスイッチにケーブル接続されます。
* DCN から ONTAP バックエンドサブネットに使用可能な IP アドレス空間範囲があります。
* ONTAPクラスタは初期化されており、クラスタ管理LIFからアクセス可能です。


.System Managerのクレデンシャル
AIDE クラスタの作成または拡張タスクを実行するには、_ストレージ管理者_権限が必要です。

.ソフトウェアの互換性
以下の文書を確認して、DCNハードウェアとソフトウェアのバージョンがONTAPクラスタと互換性があることを確認してください：

* link:../administer/upgrades-aide-summary.html["DCNソフトウェアは、ONTAPバージョンと互換性があります"]。


ノード追加操作中に、System Manager は新しいノードが次のソフトウェアバージョンと互換性があることを確認します：

* これが初めてのDCN参加の場合、ONTAPクラスターの有効バージョン（ECV）。
* DCN がすでに存在する場合は、既存の DCN クラスターのバージョン。


ノードに互換性がない場合：

* 影響を受けるDCNの横に*追加*ダイアログでエラーが表示されます。
* まず、DCN ソフトウェア（または必要に応じて ONTAP）を互換性のあるバージョンに更新する必要があります。




== データコンピューティングノードを追加する

新しいAIDEクラスターを作成するか、既存のクラスターを拡張する際に、DCNノードを追加します。

.手順
. System Managerのナビゲーション ペインで*Dashboard*を選択し、次に*Health*カードを選択します。
. 追加するノードがあることを確認し、*詳細を表示* を選択してリストを表示します。
+
リストには、まだAIDEクラスタの一部ではない検出されたノードが含まれています

. または、*クラスタ*と*概要*、*データコンピューティング*タブを選択してリストを表示することもできます。
. データ計算ページの下部で、ノードのリストの上にある *追加* を選択します。
. *データ計算ノードの追加*ダイアログで、追加したいDCNノードを選択します。
+
オプションで、個々のノードを追加する前に名前を変更できます。

. 初めてノードを追加する場合で、バックエンドサブネットが存在しない場合は、*サブネットの追加*を選択し、以下を入力します：
+
** サブネット名（内部使用用）
** サブネットアドレスとマスク
** このバックエンドネットワーク上のDCNおよびONTAPノードのIPアドレス範囲
+
System Managerは、追加されるすべてのDCNとクラスタ内のすべてのONTAPノード、およびDCNとONTAPの通信に使用される追加のクラスタレベルのフローティングIPに対して、範囲に十分な空きIPアドレスが含まれていることを検証します。



. バックエンド サブネットを追加したか、すでに存在しているかに関わらず：
+
.. 利用可能な IP アドレスを確認します。
.. 必要に応じて、*サブネットの編集* を選択し、IP 範囲を拡張します。
+
*** 範囲を拡大することしかできません。サブネットの縮小または変更はサポートされていません。
*** サブネットまたは IP 範囲を変更するには、DCN 上の基盤となる Kubernetes クラスターの再作成が必要になる場合があり、数分かかることがあります。




. 必要に応じて、次の情報を指定してData Engineサービスインターフェイスを構成します：
+
** サービスIPアドレス
** ネットワーク マスク
** ゲートウェイ（環境に必要な場合）
+
単一のIPアドレスはDCN間で負荷分散され、AI Data Engine Consoleおよび関連APIのフロントエンドアドレスとして使用されます。



. 選択したノード、バックエンド サブネット、およびData Engineサービス インターフェースの設定を確認します。
. *追加*を選択し、操作が完了するまで待ちます。System Managerは次のアクションを実行します：
+
** 選択したノードをDCNクラスタに追加します
** バックエンドネットワークをプロビジョニングし、KubernetesベースのDCNクラスタにノードを参加させます
** DCN検出用の内部メタデータを更新します


. 完了したら、*Cluster*と*Overview*を選択して確認します：
+
.. *データ計算*の下に新しいDCNがクラスターの一部として表示されます
.. すべてのノードが*正常*です
.. ダッシュボードの*Health*カードに更新されたノード数が表示されていることを確認します。
